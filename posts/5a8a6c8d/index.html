QG引擎 游戏开发指南

发表于 2018-12-14 | 更新于 2018-12-17

前言：《QG引擎 游戏开发指南》旨在帮助开发者更好的在QG游戏引擎上开发游戏，提前阅读可以有效避免“运动不平滑”、“帧动画不流程”、“画面闪烁”、“UI排版布局错误”等问题。考虑到尽可能的简单易行，本文讨论的游戏开发内容较为基础，关注点主要集中在技术实现上，而不对JS中的语法、设计模式、命名规范等进行讨论。
由于编写仓促，难免存在遗漏和错误之处。还请多包涵并指正。

游戏框架 与 主要函数

QG中的Canvas与H5的Canvas接口基本一致。设计一套简易的游戏框架往往会包含如下几个主要函数： looper， update， draw。 其中：

looper ：是游戏主循环，其通过定时器触发执行，负责管理游戏的主要逻辑。每次looper函数执行之时会依次调用所有对象的 update与 draw 函数。
update ：函数用于对游戏中的对象进行逻辑运算和数据更新（包括位移，碰撞检测，帧动画控制等）。
draw ：函数用于对游戏中的对象进行绘制，一般会通过 drawImage 之类的绘制函数实现。

如下是一段游戏框架的代码示例：

var canvas = document.getElementById("canvas");
ctx = canvas.getContext("2d");

var FPS =30;  // 因录制的视频每秒最多30帧，因此建议游戏逻辑也使用30帧每秒
var lastFrameTime = 0;
...
// 主循环
var looper = function(){
    var elapsedTime  = getTimeInMilliseconds() - lastFrameTime;
    lastFrameTime = getTimeInMilliseconds();
    ...   
    // 对象的运动和绘制
    for(var i = bulletsArray.length - 1; i >= 0; i --) { // 从后向前遍历以便删除后索引不会失效。
        bulletsArray[i].update(elapsedTime); // 使用elapseTime做位移、计算动画帧。
        bulletsArray[i].draw();

        if (bulletsArray[i].y < - 100){   // 删除移出屏幕的对象
           bulletsArray.splice(i, 1);
        }
    }
    ...
    window.setTimeout(looper, 1000 / FPS);
};
window.setTimeout(looper, 1000 / FPS)

位移运动 与 帧动画

1. 位移运动：
受运行环境、逻辑耗时等因素影响，游戏中的帧率并不是稳定的，因此 Looper 函数也并不会严格的按照setTimeout的所指定的时间被调用。
游戏开发中在对物体做位移时，应当采用 elapsedTime （也有称deltaTime）对物体位移进行运算，才能保证运动的匀速与稳定。

elapsedTime即 两次looper调用的间隔时间，其获取过程如下：

// 该函数获取当前时间
var getTimeInMilliseconds = function() { 
    var currentTime = new Date();
    return currentTime.getTime();
};

var looper = function(){
    // (当前时间 - 上次时间) --> elapsedTime
    var elapsedTime  = getTimeInMilliseconds() - lastFrameTime;
    lastFrameTime = getTimeInMilliseconds();
    ...
    obj.update(elapsedTime); // 通过elapsedTime更新对象的运动位置
    ...
}

在传入了elapsedTime的基础上，对象最终的运动位置可以通过类似如下方式实现（这里以游戏中的一颗子弹对象为例）：

// 一个子弹对象的update函数
obj.update = function(elapsedTime){
    var bulletSpeed = 32;
    obj.y -= bulletSpeed * elapsedTime / 33;   // 根据elapsedTime实现位移。
}

obj.draw = function () {
    if(bulletImg) ctx.drawImage(bulletImg, obj.x, obj.y);
}

2. 帧动画：


和位移同理， 游戏中的looper调用次数并不一一对应帧动画中的帧编号。要保证帧动画的播放速度其主要思想: 根据帧动画的播放时间计算所需播放的帧的编号。 这同样可以通过elapssedTime来实现，如下是一个爆炸动画的帧动画update逻辑：

obj.update = function(elapsedTime){
      var perFrameTime = 50;            // 每一帧图片展示的时间（毫秒）
      obj.frameShowTime += elapsedTime; // 记录动画播放的时间
      obj.frameIndex = parseInt(obj.frameShowTime / perFrameTime); // 当前帧数
      obj.frameIndex %= ANIMATION_FRAME_COUNT;
}

obj.draw = function () {
    // 根据frameIndex计算对应的帧
    if(expImgs[obj.frameIndex]) ctx.drawImage(expImgs[obj.frameIndex], obj.x, obj.y, enemyWidth, enemyHeight);
}

图片资源 与 声音资源

1. 资源加载
游戏中往往包涵大量的资源（贴图、音乐等），如果在游戏运行过程中去加载资源，往往会导致游戏的卡顿。因此建议在游戏主要逻辑开始之前加一个加载过程， 以便完成游戏中大部分资源的预先加载。
同时资源加载逻辑还应该包含进度条、百分比都UI展示，用于提示用户资源加载的进度。

如下是一个加载过程的简单示例：

const ANIMATION_FRAME_COUNT = 19
var loadCount = 0;
var loadingFinished = false;
var expImgs = new Array();

// 加载所有的帧动画
for(var i = 0; i < ANIMATION_FRAME_COUNT; ++i){
    var expImg = new Image();
    expImg.onload = function() { // 先设置监听，再进行src赋值触发实际加载
        loadCount ++;
        console.log("loading percent = " + (loadCount / ANIMATION_FRAME_COUNT))

        if (loadCount == ANIMATION_FRAME_COUNT) loadingFinished = true; // 加载完成标记
    };
    expImg.src = 'explosion' + i + '.png';
    expImgs.push(expImg);
}

var looper = function() {
    if (!loadingFinished){
        // 加载中，显示加载进度
        showLoadingPercent(loadCount / ANIMATION_FRAME_COUNT); 
    } else{
        // 加载完成，运行游戏主逻辑
        ...
    }
    window.setTimeout(looper, 1000 / FPS)
}
window.setTimeout(looper, 1000 / FPS)

2. 先设置监听再加载
由于大部分图片和声音资源都是异步加载的，因此在资源加载过程中需要设置加载完成后的监听回调。
在这个过程中需要提醒注意的是： 先设置监听，再触发加载逻辑，如下是图片和声音的示例都是正确的：

// 声音，先监听再加载
var audio = document.createElement('audio');
audio.addEventListener("canplaythrough", loadMp3);
audio.src = path;

// 图片, 先监听再加载
var img = new Image(),
img.onload = function () {
    // do something
}
img.src = src

显示布局 与 分辨率

1. 根据屏幕高宽比布局


考虑到市面上不同手机屏幕上的高宽比、分辨率都略有不同， 当考虑到布局问题时候，在 QG 中提供
window.innerWidth、window.innerHeight 两个函数获取手机屏幕的高宽信息并对画面进行布局。
如下，是对一个图片做全屏居中的绘制代码。

var gameOverImg = new Image();
gameOverImg.src = 'gameOver.png';
...
// 做居中操作时先获取屏幕的高宽
screenWidth = window.innerWidth;
screenHeight = window.innerHeight;

ctx.drawImage(
    gameOverImg, 
    screenWidth /2 - gameOverImg.width/2, 
    screenHeight /2 - gameOverImg.height/2);

2. 自定义离屏canvas高宽
不同屏幕的分辨率有所差异，可能导致绘制同样像素的物体大小不一。在 QG 中开发时， 可以通过自定义离屏canvas大小，来解决这一问题。
在自定义的离屏canvas上绘制完成后，再等比拉伸并绘制到设备上，可以保证所有手机上物体大小比例一致。（视频小游戏引擎会自行处理这一拉伸问题， 无需手动调用。）

以下是离屏canvas高宽的设置，并在不同高宽比的手机上绘制出的效果：

ctx = canvas.getContext("2d", "offscreen_richard");
canvas.width = 720; // 将离屏canvas的宽度设置为720
canvas.height = 720 * window.innerHeight / window.innerWidth; // 用屏幕高宽比算canvas的高度
...
// canvas宽度固定后，能保证这个img在所有手机上比例基本一致。
ctx.drawImage(img, 0, 0, 100, 100);

文字绘制 与 字体裁剪

QG 支持加载外部TTF 和 TTC 字体对文字进行渲染。通过font和fillStyle可以指定文字的颜色、字体、字号。如下是示例：

ctx.font="100px pingfang";
ctx.fillStyle= '#ffffff'
ctx.fillText("Small Game", window.pos_x, h/5)

注意：由于系统差异， 在 IOS 中指定的字体需要存在系统中。而对于Android 则需要将所指定的字体打包放置在”游戏工作目录” 或 “游戏工作目录\font”目录下。

由于字体文件较大（微软雅黑有14MB左右），因此在游戏上线之前建议使用字蛛等工具对字体进行裁剪，裁剪前后的字体变化如下，字体文件减小到17KB：

代码结构 与 资源路径

1. 通过qg.include实现代码
为了尽可能使得代码层次清晰， QG 建议将游戏按照模块拆分成JS文件。 为此QG提供了 qg.include() 这一函数实现文件的包含。

qg.include('cocos2d-js.js');
qg.include('src/settings.js');
qg.include('src/project.js');
qg.include('main.js');
...

2. 多JS文件下的资源路径
尽管游戏可以include多个文件，但每个文件中所引用的资源位置 都是相对于主目录 而言的 。QG中的主目录位置由客户端给定（而在浏览器中为HTML网页所在的位置）。
如下图示例中， 在 main.js 、hero.js 中加载子目录中的 hero.png 、 enemy.png 、 gameover.png 文件代码如下：

|--主目录
    | -- imageA
    |        |-- hero.png
    |        |-- hero.js  // 注意：这个JS文件在子目录中，但填写资源的路径也是相对主目录而言的。
    |
    | -- ImageB
    |        | -- enemy.png
    |
    |-- main.js // 此JS在主目录中， 路径是相对于主目录而言的。
    |-- gameOver.png

// main.js
// 此JS在主目录中， 路径是相对于主目录而言的。
var heroImg = new Image();
heroImg.src = "imageA/hero,.png";

var gameOver = new Image();
gameOver .src = "gameOver,.png";

// hero.js
// 注意：这个JS文件在子目录中，但填写资源的路径也是相对主目录而言的。
var enemyImg = new Image();
enemyImg.src = "imageB/enemy,.png";
